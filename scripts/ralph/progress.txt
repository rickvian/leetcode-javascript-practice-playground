## Codebase Patterns
- Jest configuration: Use default jest config in package.json scripts
- LeetCode file naming: [number]-[problem-name].js (e.g., 0001-two-sum.js)
- Function exports: Use ES Module syntax `export { functionName }` (NOT CommonJS module.exports)
- TreeNode problems: Define TreeNode class in solution file and export it alongside function
- LinkedList operations: Create helper ListNode class for linked list problems
- Tree problems: Export TreeNode class from solution file for use in tests
- Stub tests: All tests "fail" for stub files (expected - implementations are blank)
